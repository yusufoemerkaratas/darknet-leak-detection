### 4. Database Schema Resolution (Sprint 4 Integration Fix)
During the integration check, we identified that the `datenleck_collector` container was failing with a `psycopg2.errors.UndefinedColumn` error because database columns `error_count_transient`, `error_count_persistent`, and `average_latency_ms` were defined in python models but missing from Alembic migrations.

We autogenerated a new Alembic migration [c360b11bc6e4_add_error_tracking_and_latency_columns_](file:///home/yusufkaratas/Documents/ki/darknet-leak-detection/backend/alembic/versions/c360b11bc6e4_add_error_tracking_and_latency_columns_.py), successfully ran the upgrade (`alembic upgrade head`), and confirmed the live collector scheduler picked up and ran ransomware site crawling without errors.
