# Live Demo Script - Darknet Leak Detection System

This document outlines the step-by-step live demonstration scenario designed for the final presentation. It spans **10 to 15 minutes** and guides the project team through a live system walk-through.

---

## Preparation (2 minutes before presentation)
Ensure all Docker containers are running and operational:
```bash
docker compose up -d
```
Verify that the services are active:
```bash
docker compose ps
```
*Expected: `datenleck_backend`, `datenleck_collector`, `datenleck_frontend`, `datenleck_postgres`, and `datenleck_tor` should all show a status of `Up` / `healthy`.*

---

## Step 1: Health & API Verification (2 Minutes)
Start the presentation by proving the backend service and database connection are operational using `curl`.

1. Run the health check:
   ```bash
   curl http://localhost:62001/health
   ```
   *Expected Response:* `{"status":"ok"}`
2. Run the system metrics check:
   ```bash
   curl http://localhost:62001/stats
   ```
   *Expected Response:* Returns a JSON with `database_status: "online"`, total record count (e.g. `21896`), and records count per source.
3. *Presenter Dialogue:* *"We verify that our FastAPI backend service is healthy and our PostgreSQL database is connected with over 21,000 processed leak documents.*"

---

## Step 2: Open Dashboard & Overview (3 Minutes)
Demonstrate the main analyst interface.

1. Open your browser and navigate to:
   ```text
   http://localhost:62000
   ```
2. **Dashboard Shell:** Show the modern, responsive dark-mode dashboard layout. Explain the layout blocks:
   - **Overview Stats Cards:** Total Findings, Critical Alerts, Reviewed Findings, and Monitored Companies.
   - **Timeline Chart:** Displays the trend of findings collected over time (7d, 30d, 365d).
   - **Feed Metrics:** Shows breakdown by severity (Critical, High, Medium, Low) and breakdown by data sources (e.g. Paste Site, ransomware_sites).
   - **Sidebar status cards:** Shows live status of AI engine and system health.
3. *Presenter Dialogue:* *"This is the analyst cockpit. It provides real-time metric visualization, showing alert trends, severity distributions, and system health status. It is optimized to stay responsive even under thousands of loaded records.*"

---

## Step 3: Filtering & Detailing Findings (3 Minutes)
Showcase how analysts can search, filter, and inspect findings.

1. **Filtering:** Use the UI filters on the sidebar or company ranking cards to filter findings by company (e.g., click on **"Zoom"** or **"Salesforce"**).
2. **Detailed View:** Click on a finding candidate from the recent feed list (e.g. **Krybit Leak Site Dump** or **Payload Leak Site Dump**).
   - Show the detail Modal that slides or pops open.
   - Walk through the structured details: matched company, leak raw URL, estimated dump size, published date, and evidence list.
3. **Deterministic Explanation:** Show the threat explanation that was generated by the parser pipeline.
4. *Presenter Dialogue:* *"When an analyst clicks on a finding, the system presents complete contextual evidence (onion urls, size, matched patterns) along with recommended remediation steps like credential rotation.*"

---

## Step 4: AI Analysis On-Demand (2 Minutes)
Demonstrate the LLM integration in action.

1. In the finding detail Modal, locate the **"Analyze with AI"** button.
2. Click the button to trigger a live call to the LLM backend endpoint:
   ```text
   POST /dashboard/findings/{finding_id}/llm-analysis
   ```
3. Watch the explanation update live with the AI's parsed analysis summarizing the threat (or showing fallback states gracefully if rate limits are hit).
4. *Presenter Dialogue:* *"Instead of analyzing 21,000 findings automatically and hitting rate-limits, our system implements an 'On-Demand' approach. The analyst selects a record, hits 'Analyze with AI', and queries the LLM for a targeted threat explanation.*"

---

## Step 5: Incident Review Workflow & Custom Actions (2 Minutes)
Demonstrate how the analyst takes action.

1. In the open finding detail view, locate the Status control.
2. Change the status from **"Not Reviewed"** to **"Reviewed"** or **"False Positive"**.
3. Close the modal and show that the dashboard stats (Reviewed count) update immediately.
4. *Presenter Dialogue:* *"The analyst can dismiss false positives or escalate high-risk cases. Marking a finding updates the database state immediately, feeding back into the detection calibration model.*"

---

## Step 6: Live Crawler Trigger & Conclusion (2 Minutes)
Wrap up the live demonstration.

1. Explain that the collector runs in the background (`datenleck_collector` container) scheduling darknet scrapes over Tor.
2. Conclude the demo by summarizing the key achievements:
   - Automated collection over Tor.
   - High-throughput ingestion (900k+ records/min capability).
   - Strict pattern-matching and company mapping to eliminate false positives.
   - On-demand AI enrichment.
3. *Presenter Dialogue:* *"To summarize, LeakGuard provides a resilient, low-latency early warning system that protects corporate assets against credentials, database dumps, and active darknet threats. Thank you for your time.*"

---
